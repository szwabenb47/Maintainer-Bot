# Contributing

Thanks for considering a contribution to Maintainer-Bot.

## Development Setup

Maintainer-Bot uses Node.js 20 and has no runtime npm dependencies.

```bash
npm test
npm run lint
```

## Pull Requests

Please keep pull requests focused. Include tests for behavior changes and update
the README or docs when a user-facing workflow changes.

Good pull requests usually include:

- a short problem statement;
- a clear description of the behavior change;
- tests for the changed behavior;
- notes about privacy or security impact when relevant.

## Maintainer Principles

- Keep automation explainable.
- Keep maintainers in control of final decisions.
- Prefer safe defaults for public repositories and forked pull requests.
- Avoid collecting more repository context than a workflow needs.
