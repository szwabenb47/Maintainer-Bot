# Maintainer-Bot

Maintainer-Bot is a lightweight GitHub Action for open-source maintainer
workflows. It helps with issue triage, pull request review summaries, and
release preparation while keeping final decisions in human hands.

The action works in two modes:

- deterministic mode, which needs no API key and produces rule-based guidance;
- AI-assisted mode, which can use the OpenAI API for richer maintainer notes.

## Project Status

This repository is an early public MVP. The goal is to provide useful,
auditable automation for maintainers of small and mid-sized OSS projects before
expanding into deeper review and security workflows.

## Features

- Issue triage with suggested labels and missing-information checks.
- Pull request summaries with changed-file overview, risk level, and review
  checklist.
- Release notes review with follow-up checklist.
- Dry-run mode for public repositories and pull requests from forks.
- No runtime npm dependencies.
- Optional OpenAI API integration through the Responses API.

## Quick Start

Add this workflow to `.github/workflows/maintainer-bot.yml`:

```yaml
name: Maintainer Bot

on:
  issues:
    types: [opened, edited, reopened]
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]
  release:
    types: [published, edited]

permissions:
  contents: read
  issues: write
  pull-requests: read

jobs:
  assist:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: szwabenb47/Maintainer-Bot@main
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          dry-run: "true"
          post-comment: "false"
```

To enable AI-assisted summaries, add an `OPENAI_API_KEY` repository secret and
set `dry-run` to `false`:

```yaml
      - uses: szwabenb47/Maintainer-Bot@main
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          dry-run: "false"
          post-comment: "true"
```

## Inputs

| Input | Default | Description |
| --- | --- | --- |
| `github-token` | `${{ github.token }}` | Token used to read PR metadata and optionally post comments. |
| `openai-api-key` | empty | Optional API key for AI-assisted summaries. |
| `model` | `gpt-5.5` | OpenAI model used when AI-assisted mode is enabled. |
| `mode` | `auto` | `auto`, `issue`, `pull_request`, or `release`. |
| `dry-run` | `true` | When true, the action avoids OpenAI calls and comments. |
| `post-comment` | `false` | When true and `dry-run` is false, posts the report as an issue or PR comment. |
| `max-diff-files` | `20` | Maximum number of PR files included in the prompt/report context. |
| `max-diff-chars` | `12000` | Maximum diff characters included in AI-assisted context. |

## Outputs

| Output | Description |
| --- | --- |
| `summary` | Markdown summary generated by the action. |
| `mode` | Mode selected for the current event. |

## Security and Privacy

Maintainer-Bot is designed for transparent OSS workflows:

- dry-run mode is the default;
- comments are opt-in;
- OpenAI API usage is opt-in;
- the action sends only event metadata and selected PR diff context when
  AI-assisted mode is enabled;
- maintainers should avoid enabling AI-assisted mode for repositories that may
  expose secrets in diffs or issue bodies.

See [docs/privacy-and-security.md](docs/privacy-and-security.md) for details.
