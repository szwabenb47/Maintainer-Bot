# Application Notes

These notes help describe Maintainer-Bot accurately when applying to OSS
programs or asking other maintainers to try it.

## Repository Qualification Summary

Maintainer-Bot provides open-source maintainer automation for GitHub: issue
triage, pull request review summaries, release checklists, and optional
AI-assisted maintainer notes. It is public, tested, and designed to reduce
routine review and triage work while keeping final decisions human-approved.

## API Credit Usage Summary

API credits would be used for maintainer workflows: issue triage, pull request
review summaries, release note preparation, regression-risk checks, and
security-oriented review notes. Automated output is advisory and reviewed by a
human maintainer before action is taken.

## Evidence to Collect Before Applying

- Public repository URL.
- CI status and test coverage signals.
- Releases or version tags.
- Example issues or pull requests where the action produced useful output.
- Any public repositories using the action.
- Stars, forks, contributors, downloads, or other real usage metrics.

Use accurate numbers only. Do not overstate adoption, maintainer role, or
repository impact.
